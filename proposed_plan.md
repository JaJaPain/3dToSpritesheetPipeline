# Preserve TRELLIS Character Silhouette & Segment UV Layout

This plan changes the Blender decimation and baking process to keep the actual unique character shape/silhouette generated by TRELLIS. Instead of baking colors onto a blocky cylinder template, we will decimate the TRELLIS mesh directly, segment its faces into body parts (head, torso, legs, feet) using coordinate limits, map those segments to the corresponding quadrants of the 4-color `UVGuide.png`, and bake the high-poly TRELLIS textures onto this restructured UV map.

## User Review Required

> [!NOTE]
> **Preserving Character Shape:**
> Your character's unique shape (hair, face structure, custom clothing contours) will now be fully preserved in the exported FBX file. The only change is that its UV map coordinates will be cleanly organized into the four quadrants of your `UVGuide.png`.

> [!IMPORTANT]
> **Segmentation Height Thresholds:**
> We will segment the character mesh into four regions based on relative heights (Z-axis percentages) and arm-width offsets (X-axis):
> - **Head/Neck/Hands (Skin)**: Top $18\%$ of the model's height, plus vertices that are outer-extremities along the X-axis (hands).
> - **Torso/Arms (Shirt)**: Upper-middle section ($50\%$ to $82\%$ of height).
> - **Hips/Legs (Pants)**: Lower-middle section ($12\%$ to $50\%$ of height).
> - **Feet (Shoes)**: Bottom $12\%$ of height.
> This works automatically for standard humanoid T-pose or A-pose models.

## Proposed Changes

### Blender Headless Core Engine

#### [MODIFY] [blender_headless_core.py](file:///c:/CodingProjects/3D_To_Spritesheet/core/blender_headless_core.py)

We will modify the script to perform these steps:
1. **Load original TRELLIS GLB**: Import the high-poly mesh.
2. **Duplicate and Decimate**:
   - Create a copy of the TRELLIS mesh.
   - Apply a Decimate modifier to the copy to reduce it to a target polycount (e.g. 15,000 faces) suitable for rigging and game engine performance.
3. **Segment and Unwrap the Decimated Mesh**:
   - Calculate the bounding box of the decimated mesh to get the height $H$ and width $W$.
   - Group the mesh's faces into 4 list categories:
     - **Skin (Top-Left UV)**: $Z \ge Z_{min} + 0.82 H$ (Head) OR (hands check: $Z$ is between $0.35 H$ and $0.65 H$ and $|X| \ge 0.25 W$).
     - **Shirt (Top-Right UV)**: $Z$ is between $0.50 H$ and $0.82 H$ (excluding hands).
     - **Pants (Bottom-Left UV)**: $Z$ is between $0.12 H$ and $0.50 H$.
     - **Shoes (Bottom-Right UV)**: $Z < Z_{min} + 0.12 H$.
   - Segment by segment:
     - Select the faces in the segment.
     - Execute Blender's Smart UV Project on the selected faces.
     - Normalize and map the UV coordinates of those face loops to fit within the designated $0.5 \times 0.5$ quadrant:
       - Skin $\rightarrow$ scale by $0.48$, offset to $(0.01, 0.51)$
       - Shirt $\rightarrow$ scale by $0.48$, offset to $(0.51, 0.51)$
       - Pants $\rightarrow$ scale by $0.48$, offset to $(0.01, 0.01)$
       - Shoes $\rightarrow$ scale by $0.48$, offset to $(0.51, 0.01)$
4. **Bake Diffuse Colors**:
   - Set the decimated mesh as active and the high-poly TRELLIS mesh as selected.
   - Run Cycles Selected-to-Active bake to transfer all textures and vertex colors onto the newly formatted UV coordinates.
5. **Export the Decimated Character FBX**:
   - Delete the high-poly source mesh.
   - Export the decimated character mesh with the clean UVs and baked texture to `workspace/02_optimized_fbx/`.
